# BTC EMA20 失衡衰竭交易策略 V0.1

## 1. 策略定位

这是第一版可回测原型，不是最终交易系统。

核心目标：把主观交易语言转成机器可以执行的条件，先跑出样本，再根据实际回测结果继续优化。

适用场景：

- 品种：BTC
- 周期：3分钟为主
- 风格：失衡后的反转 / 衰竭交易
- 均线核心：EMA20
- 目标：少做横盘中间位置，多等单边失衡后的衰竭确认

## 2. 核心交易思想

你的原始打法可以拆成六个内部指标，而不是直接用一个总分粗暴判断：

1. Age：价格沿 EMA20 单边运行的时间是否足够长。
2. Extension：价格是否明显远离 EMA20，形成失衡。
3. Flatten：EMA20 是否从很陡逐渐走平，说明原方向动能减弱。
4. Touch：价格是否重新回到 EMA20 附近，而不是在很远的位置追单。
5. Rejection：是否出现上影线/下影线，代表冲击失败或承接失败。
6. Confirm：是否出现和原趋势相反的“不好K线”。

V0.1 将这六个指标量化为 0-100 分。

## 3. 做空逻辑

做空不是看到上涨就空，而是要求：

- 价格在 EMA20 上方运行足够久；
- 前面有明显向上失衡；
- EMA20 过去很陡，但最近斜率开始走弱；
- 价格重新靠近 EMA20；
- 出现偏空确认K线；
- 当前不是来回穿 EMA20 的震荡盘。

## 4. 做多逻辑

做多是做空逻辑的镜像：

- 价格在 EMA20 下方运行足够久；
- 前面有明显向下失衡；
- EMA20 过去很陡，但最近斜率开始走弱；
- 价格重新靠近 EMA20；
- 出现偏多确认K线；
- 当前不是来回穿 EMA20 的震荡盘。

## 5. 风控逻辑

V0.1 默认使用结构止损：

- 做多：止损放在近期低点下方，再加 ATR 缓冲。
- 做空：止损放在近期高点上方，再加 ATR 缓冲。
- 止盈：默认 3R。

后续可以继续测试 2R、3R、5R、分批止盈、移动止损。

## 6. 横盘过滤

V0.1 避免横盘的方式：

- 统计最近一段时间价格穿越 EMA20 的次数。
- 如果穿越次数太多，认为是震荡盘，过滤。
- 同时要求最近波动区间不能太小，避免在死水盘里开单。

## 7. 当前版本的问题

V0.1 只是骨架，重点是能跑回测。它还没有解决以下问题：

- 没有真正识别高级别支撑/阻力；
- 没有识别流动性扫损；
- 没有加入成交量衰竭；
- 没有加入 RSI/MACD 背驰；
- 没有区分“第一次急跌反弹不能做”和“二次确认可以做”；
- 没有动态判断主力诱多/诱空；
- 没有根据不同波动环境自动切参数。

## 8. 下一版计划

V0.2 建议加入：

1. 二次确认模块：避免急涨急跌后的第一下反抽直接介入。
2. 背驰模块：加入 RSI 或成交量背驰。
3. 高低点结构模块：识别 HH/HL/LH/LL。
4. 盘口方向过滤：只做主方向最明确的一边。
5. 回测统计模板：记录胜率、盈亏比、最大回撤、连续亏损次数、盈利因子。

## 9. 使用方法

1. 打开 TradingView。
2. 进入 BTCUSDT 3分钟图。
3. 打开 Pine Editor。
4. 粘贴 `tradingview/btc_ema20_imbalance_v0_1.pine`。
5. 点击 Add to chart。
6. 打开 Strategy Tester 查看回测。
7. 把回测结果、明显错单截图、明显好单截图发回来，继续迭代 V0.2。
